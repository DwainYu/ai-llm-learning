### 报错1
``` bash
wsl: 检测到 localhost 代理配置，但未镜像到 WSL。NAT 模式下的 WSL 不支持 localhost 代理。

```
这个报错是 WSL 的**网络隔离机制**导致的——Windows 检测到本地代理，但 NAT 模式下无法将 `localhost` 映射到 WSL 环境，导致代理在 WSL 内失效。
### 核心原因
WSL 默认使用 NAT 网络，其 `127.0.0.1` 是独立环回接口，与 Windows 主机不互通；而代理默认绑定 `localhost`，WSL 无法共享该连接，因此触发提示。

开启镜像模式（推荐，一劳永逸）
1. **检查环境**：确认 Windows 11 22H2+、WSL 1.2.0+（`wsl --version` 查看）。
2. **配置文件**：在 Windows 用户目录（`C:\Users\<用户名>`）新建/编辑 `.wslconfig`，添加：
```
[wsl2]
networkingMode=mirrored  # 镜像模式
autoProxy=true           # 自动同步代理
dnsTunneling=true
firewall=true
```
3. **生效配置**：管理员 PowerShell 执行：
```powershell
wsl --shutdown
wsl --update  # 可选，确保最新
wsl
```
重启后 WSL 与 Windows 共享网络，`localhost` 代理自动可用。

### 微信配置




```
How should direct messages be authorized?
 → Use DM pairing approval (recommended) 
 Allow all direct messages 
 Only allow listed user IDs 
 Disable direct messages
```

| 选项                             | 含义                        | 适用场景          |
| ------------------------------ | ------------------------- | ------------- |
| **DM pairing approval** ⭐推荐    | 第一次发消息的人需要你手动"批准"，之后就能正常聊 | 既安全又灵活，适合个人微信 |
| **Allow all direct messages**  | 谁都能直接和你的机器人聊              | 适合公开客服机器人     |
| **Only allow listed user IDs** | 只有你提前写进配置的用户 ID 才能聊       | 适合只给自己用的场景    |
| **Disable direct messages**    | 禁止私聊，只在群里回复               | 不太适合微信场景      |
